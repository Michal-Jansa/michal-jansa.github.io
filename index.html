Power BI - Portfolio Performance Dashboard


This dashboard provides a clear, data-driven overview of an investment portfolio for 2025, including year-to-date performance, benchmark comparison, drawdown, and other key metrics. It also addresses a common limitation in reporting for actively managed portfolios: results are often presented only at the portfolio level, offering limited insight into how the individual investment strategies within the portfolio perform.
Instead, this dashboard presents results at both the portfolio and strategy level by breaking down performance and metrics for each individual strategy. This helps users understand which investment strategies contribute most to total returns, which underperform, and how each one affects overall risk.
Built on real personal investment data from 2025, the dashboard features an easy-to-use interactive allocation tool. Users can adjust the portfolio's starting capital and redistribute it across four investment strategies to simulate alternative outcomes. This allows investors to explore "what-if" scenarios and gain practical insights into how different allocations could improve overall performance or reduce risk of the whole portfolio.
This project was created in Power BI.

Skills Demonstrated

- Building a simple relational Data Model in Power BI (one to many relationship, one to one relationship)

- Creating parameters for user input and interaction (numeric and field parameters)

- Writing a large set of complex DAX measures to produce accurate dynamic calculations and outputs

- Designing the dashboard layout by adding charts and slicers with user-adjustable sliders

- Linking DAX measures to the corresponding visual elements

- Formatting visuals with appropriate colors, fonts and conditional formatting

- Adding additional interactivity through tooltip buttons and Power BI bookmarks

MS Excel - Operating Costs Report


This project showcases a financial controlling report summarizing a government institution's internal operating costs for 2025, compared with the previous year. The data is broken down by region, quarter, department type and cost category. The source dataset was exported from a SAP CO (Controlling) module and consists of a small fraction of real records which were heavily randomized. Additional data was fully simulated to both reduce dataset size and protect confidential financial information.
The report is intentionally designed to be visually clear and presentation friendly. It features intuitive navigation links, cleanly formatted Pivot Tables and Pivot Charts, and interactive slicers that allow managers to explore the data with a single click. The goal of the report is to make financial insights easily accessible even to non technical staff, enabling them to evaluate cost efficiency, staffing needs or overall spending across the organization.
This project was created using only the base version of Microsoft Excel without any add ins or specialized features, since Excel is a standard tool available in almost every workplace.

Skills Demonstrated

- Randomizing, simulating, and structuring a dataset in Excel (RAND, RANDBETWEEN, IF)

- Categorizing operating costs using reference tables (RIGHT, LEFT, VLOOKUP)

- Designing and formatting PivotTables and PivotCharts (calculated fields, custom PivotTable styles, conditional formatting)

- Adding slicers to filter corresponding PivotTables and PivotCharts

- Creating an introductory menu page and navigation buttons for easier use (background image, responsive buttons with hyperlinks)

For a detailed technical description and access to the file, visit the project's GitHub page. To download the file directly, click here.
No additional software beyond standard Excel is required.

Power Query and Power Pivot - Statistical report


This report presents statistical data on employees in Prague who received statutory sickness benefits during a selected year, broken down by age, gender, city district, and other factors. All figures are shown in relation to the total number of employed persons registered in the capital during the same period. The source dataset is based on a partial extract of real records which have been further anonymized and randomized to ensure the data is presented safely and without any sensitive personal information.
The resulting analysis provides insight into the structure of the Prague workforce and highlights which groups of employees make use of statutory sickness benefits, for how long, and for which health reasons. It serves as a data foundation for materials produced by a national statistical institution, relevant ministries or government departments - whether for official documents or as direct support for potential policy decisions.
This project was created in Microsoft Excel using advanced data processing tools of Power Query and Power Pivot.

Skills Demonstrated

- Data import, cleaning, and transformation in Power Query (data type adjustments, filtering, error handling, column removal, loading selected queries into the Data Model)

- More advanced Power Query operations (merging queries using Left Join, creating custom columns with M language, adding conditional logic)

- Building a simple relational Data Model in Power Pivot (one to many relationships)

- Creating calculated columns in Power Pivot using DAX (ROUNDDOWN, DATEDIFF, SWITCH, RELATED)

- Creating basic DAX measures (DISTINCTCOUNT, COUNTROWS, CALCULATE, MEDIAN)

- Designing Pivot Tables to present the final results
